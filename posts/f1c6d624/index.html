ps 进程查看器

Linux中的ps命令是Process Status的缩写。ps命令用来列出系统中当前运行的那些进程。ps命令列出的是当前那些进程的快照，就是执行ps命令的那个时刻的那些进程，如果想要动态的显示进程信息，就可以使用top命令。

要对进程进行监测和控制，首先必须要了解当前进程的情况，也就是需要查看当前进程，而 ps 命令就是最基本同时也是非常强大的进程查看命令。使用该命令可以确定有哪些进程正在运行和运行的状态、进程是否结束、进程有没有僵死、哪些进程占用了过多的资源等等。总之大部分信息都是可以通过执行该命令得到的。

ps 为我们提供了进程的一次性的查看，它所提供的查看结果并不动态连续的；如果想对进程时间监控，应该用 top linux下的任务管理器 工具。

注：kill -9 命令用于杀死进程。

ps 进程查看器

linux上进程有5种状态:

运行(正在运行或在运行队列中等待)
中断(休眠中, 受阻, 在等待某个条件的形成或接受到信号)
不可中断(收到信号不唤醒和不可运行, 进程必须等待直到有中断发生)
僵死(进程已终止, 但进程描述符存在, 直到父进程调用wait4()系统调用后释放)
停止(进程收到SIGSTOP, SIGTSTP, SIGTTIN, SIGTTOU信号后停止运行运行)
ps工具标识进程的5种状态码:

  1. D 不可中断 uninterruptible sleep (usually IO)
  2. R 运行 runnable (on run queue)
  3. S 中断 sleeping
  4. T 停止 traced or stopped
  5. Z 僵死 a defunct (”zombie”) process

    命令参数

  6. a 显示所有进程
  7. -a 显示同一终端下的所有程序
  8. -A 显示所有进程
  9. c 显示进程的真实名称
  10. -N 反向选择
  11. -e 等于“-A”
  12. e 显示环境变量
  13. f 显示程序间的关系
  14. -H 显示树状结构
  15. r 显示当前终端的进程
  16. T 显示当前终端的所有程序
  17. u 指定用户的所有进程
  18. -au 显示较详细的资讯
  19. -aux 显示所有包含其他使用者的行程
  20. -C<命令> 列出指定命令的状况
  21. –lines<行数> 每页显示的行数
  22. –width<字符数> 每页显示的字符数
  23. –help 显示帮助信息
  24. –version 显示版本显示

    输出列的含义

  25. F 代表这个程序的旗标 (flag)， 4 代表使用者为 super user
  26. S 代表这个程序的状态 (STAT)，关于各 STAT 的意义将在内文介绍
  27. UID 程序被该 UID 所拥有
  28. PID 进程的ID
  29. PPID 则是其上级父程序的ID
  30. C CPU 使用的资源百分比
  31. PRI 这个是 Priority (优先执行序) 的缩写，详细后面介绍
  32. NI 这个是 Nice 值，在下一小节我们会持续介绍
  33. ADDR 这个是 kernel function，指出该程序在内存的那个部分。如果是个 running的程序，一般就是 “-“
  34. SZ 使用掉的内存大小
  35. WCHAN 目前这个程序是否正在运作当中，若为 - 表示正在运作
  36. TTY 登入者的终端机位置
  37. TIME 使用掉的 CPU 时间。
  38. CMD 所下达的指令为何
